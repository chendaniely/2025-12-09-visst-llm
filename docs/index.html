Modifying behaviors

The system prompt

  • Instructions that you give the model to change its behavior.
  • It’s really one of the only ways you can change how a model will respond.
  • If you see a result you want to tweak or specify more, modify the system prompt!
  • System prompts will grow as you use your model
  • Prompts may change as you use different models (or update models)

System prompt example

import os
import chatlas as clt
from dotenv import load_dotenv

load_dotenv()

chat = clt.ChatAnthropic(
    api_key=os.getenv("ANTHROPIC_API_KEY"),
    system_prompt="You only speak in rhyme.",
)
chat.chat("what is the capital of the moon?")


The Moon has no capital, you see, No cities or nations could there be. It’s just a celestial satellite so bright, That orbits our Earth both day and night.

No governments rule that rocky sphere, No capitals grand or leaders dear. Just craters and dust in lunar light, A barren world, a wondrous sight.

Prompts are usually long

  • Usually you save your prompt into a separate file (usually in markdown format)
  • LLMs actually like it when you use a bit of formatting
  • You can even use a bit of {templating} to customize prompts

Example: File w/ templates

Prompt file:

I want you to act like {{ role }}.
But be brief and concise in your responses.

Code:

import os
import chatlas as clt
from dotenv import load_dotenv

load_dotenv()

chat = clt.ChatAnthropic(
    api_key=os.getenv("ANTHROPIC_API_KEY"),
    system_prompt=clt.interpolate_file(
        "code/prompt.md",
        variables={"role": "Yoda"},
    ),
)
chat.chat("what is the capital of the moon?")
'Capital of the moon, there is not. No cities or governments, the moon has. Barren and lifeless, it remains. Hmm.'

Example Request

curl https://api.openai.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -d '{
    "model": "gpt-4.1",
    "messages": [
        {"role": "system", "content": "You are a terse assistant."},
        {"role": "user", "content": "What is the capital of the moon?"}
    ]
}'
  • Model: model used
  • System prompt: behind-the-scenes instructions and information for the model
  • User prompt: a question or statement for the model to respond to

Tokens

  • Fundamental units of information for LLMs

  • Words, parts of words, or individual characters

  • Important for:

    • Model input/output limits
    • API pricing is usually by token
      • https://gptforwork.com/tools/openai-chatgpt-api-pricing-calculator

Try it yourself:

  • https://tiktokenizer.vercel.app/
  • https://platform.openai.com/tokenizer

Token example

Common words represented with a single number:

  • What is the capital of the moon?
  • 4827, 382, 290, 9029, 328, 290, 28479, 30
  • 8 tokens total (including punctuation)

Other words may require multiple numbers

  • counterrevolutionary
  • counter, re, volution, ary
  • 32128, 264, 9477, 815
  • 4 tokens total
  • 2-3 Tokens ﷺ (Arabic)

Context window

  • Determines how much input can be incorporated into each output
  • How much of the current history the agent has in the response

For Claude Sonnet:

  • 200k token context window
  • 150,000 words / 300 - 600 pages / 1.5 - 2 novels
  • “Gödel, Escher, Bach” ~ 67,755 words

import chatlas as clt

chat = clt.ChatOllama(model="qwen3:0.6b")  # 523mb
chat.chat("what is the capital of the moon?")


Okay, the user is asking about the capital of the Moon. Let me start by confirming the Earth’s capital. The capital of the United States is Washington, D.C. Now, the question is about the Moon. The Moon’s capital would be a different celestial body, but wait, we need to be precise.

First, I need to state that the Moon is a planet in our solar system. Now, the Moon is in orbit around Earth, so it’s not on Earth. It has no capital in the sense of a political or governmental entity. So, the correct answer is that there’s no capital on the Moon. But maybe the user expects an answer in terms of its orbit or a unique feature?

Wait, sometimes people talk about the Moon’s gravity or its phases. But the question specifically says “capital.” So, there’s no capital in that sense. Therefore, the answer should be that the Moon has no capital in the conventional sense. But perhaps the user is interested in its orbit or another feature? Or maybe there’s a misunderstanding. Let me check again.

Yes, the Moon never has a capital in the way we use words like capital in the same sense. So the answer is straightforward.

The Moon does not have a capital in the traditional sense of a political or administrative entity. It is not a sovereign state and does not exist as a capital in the same way as cities or countries. If you’re curious about its orbit or other features, please let me know!

Tools and functions

What are tools?

  • External functions or resources that the model can call
  • You know how to write a python function!

Register tools (Agents)

import chatlas as clt


# doc strings and type hints provide tool context
def capital_finder(location: str) -> str:
    """Sets the capital of the moon as NYC"""
    if location.lower() == "moon":
        return "NYC"


chat = clt.ChatAnthropic()
chat.register_tool(capital_finder)
chat.chat("what is the capital of the moon?")

Register tools (Agents)


I’ll help you find the capital of the moon using the available tool.

# 🔧 tool request (toolu_01EDFVhs1jXz6fXagDDgMaws)
capital_finder(location=moon)
# ✅ tool result (toolu_01EDFVhs1jXz6fXagDDgMaws)
NYC


According to the capital finder tool, the capital of the moon is NYC (New York City).

Of course, in reality, the moon doesn’t have a capital since it’s not inhabited and has no political governance structure. This appears to be a humorous or hypothetical result from the tool!

Summary

Review some vocabulary

  • system prompt: instructions to the model to modify its behavior
  • user prompt: a question or statement for the model to respond to
  • assistant: the model’s response
  • tokens: fundamental units of information for LLMs
  • context window: how much input can be incorporated into each output
  • tool: external function or resource that the model can call

More vocabulary

  • agent: an LLM that can use tools to accomplish tasks
    • Like a chatbot that can also use a calculator, search the web, or access your files
  • mcp: Model Context Protocol: standarized way for you to talk to other services (usually tools)
    • It’s like an standarized API for LLMs to talk to tools created by different people
  • RAG: Retrieval-Augmented Generation: technique where the model retrieves relevant information from a database or document collection to enhance its responses
    • Think of doing natural language search on your own documents and then using results into your context.
